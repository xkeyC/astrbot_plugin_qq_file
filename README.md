# QQ文件助手插件

让 LLM 拥有感知 QQ 群资料和文件的能力，支持群文件、群相册、群成员分页查询及文件下载链接等操作。支持监听群文件上传并自动触发处理。

## 功能

- 统一查询群文件和群相册（支持文件夹、相册媒体及分页游标）
- 按昵称或群名片搜索并分页查询群成员，明确标注群主、管理员和普通成员身份
- 获取文件直接下载链接
- 搜索群文件
- 支持群聊和私聊
- **群文件上传自动监听与处理**（v3.0 新增）
- **多模板组配置，自上而下匹配**（v3.1 新增）
- 纯 LLM Tools 设计，无需记忆命令

## 使用方式

本插件**仅提供 LLM Tools**，所有功能通过自然语言与 AI 对话即可使用。

### 可用 LLM Tools

| 工具名称 | 功能 | 适用场景 |
|---------|------|---------|
| `qq_file` | 统一查询群文件或群相册，每个结果含 `source` | 用户说“看看群文件”“查看群相册”“列出这个相册的照片”等 |
| `qq_group_members` | 搜索并分页查询群成员及身份，可按身份筛选 | 用户说“搜索昵称为小明的成员”“查看管理员”“下一页群成员”等 |
| `qq_get_file_download_url` | 获取群文件的直接下载链接 | 用户说“下载这个文件”“给我文件链接”等 |
| `qq_search_files` | 搜索群文件 | 用户说“找一下xxx文件”“搜索pdf文件”等 |
| `qq_get_file_info` | 获取群文件详细信息 | 用户想查看文件元数据（大小、上传者等） |
| `qq_request_file_upload` | 请求上传文件到群文件 | 模型已准备好一个文件并需要上传到群 |
| `qq_delete_files` | 批量删除群文件 | 用户明确要求删除一个或多个群文件 |
| `qq_delete_folder` | 删除群文件夹 | 用户明确要求删除群文件夹及其中内容 |

所有 LLM Tool 名称均以 `qq_` 开头，便于模型识别其 QQ 能力边界。

### `qq_file` 来源与分页

- `source=group_file`（默认）：查询群文件。可使用 `folder_id` 浏览文件夹。
- `source=group_album` 且不传 `album_id`：查询群相册；相册映射为 `type=folder`、`folder_type=album`。
- `source=group_album` 且传 `album_id`：查询相册媒体；照片/视频映射为 `type=file`、`file_type=album_media`。
- 每个文件或文件夹条目都有 `source`，值为 `group_file` 或 `group_album`。
- 群相册返回 `has_more` 和 `next_cursor`；继续查询时原样传回 `cursor`。

### `qq_group_members` 分页

- `page` 从 1 开始，`page_size` 范围为 1–100。
- `keyword` 对昵称和群名片进行不区分大小写的模糊搜索，搜索后再分页。
- `role` 可选 `owner`、`admin` 或 `member`；返回项同时包含英文 `role` 和中文 `role_name`。
- 结果按群主、管理员、普通成员排序，并返回全群 `role_counts` 统计。

**群聊场景：**

- "查一下群主和管理员"

## 配置

在 AstrBot 控制台配置以下选项：

### 基础配置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `access_mode` | string | 访问控制模式：`blacklist` (黑名单) / `whitelist` (白名单) |
| `group_list` | string | 群号列表，英文逗号分隔，如 `123456,789012` |
| `max_file_list_limit` | int | 最大文件列表返回数量 (1-100) |
| `enable_file_search` | bool | 是否启用文件搜索功能 |

### 自动处理配置

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `enable_auto_process` | bool | 是否启用文件自动处理 |
| `auto_process_templates` | list | 自动处理模板组列表 |

### 模板组配置说明

每个模板组包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_ids` | textarea | 群号码列表，一行一个，留空表示匹配所有群 |
| `file_patterns` | textarea | 文件名正则表达式，一行一个，留空表示匹配所有文件 |
| `skills` | list | 建议使用的技能列表，None 或空数组不注入提示，非空时会在提示中建议 LLM 使用指定技能 |

### 模板匹配规则

1. **自上而下匹配**：模板组按配置顺序从上到下依次匹配
2. **首次匹配生效**：文件匹配到第一个符合条件的模板后，立即触发处理，不再继续匹配
3. **群号匹配**：如果模板配置了群号，只在指定群内生效；留空则所有群都匹配
4. **文件名匹配**：文件名需匹配任意一个正则表达式；留空则匹配所有文件

### 访问控制说明

- **Blacklist 模式**: `group_list` 中的群**无法**使用文件功能（默认）
- **Whitelist 模式**: 只有 `group_list` 中的群**可以**使用文件功能
- **私聊**: 在 blacklist 模式下，私聊默认允许（除非用户在黑名单）；whitelist 模式下私聊不允许

### 自动处理示例

**场景1：数据分析群，自动分析表格文件**

模板组配置：
- `group_ids`: `123456789`
- `file_patterns`: `.*\.csv$` 和 `.*\.xlsx$`（分行填写）
- `skills`: 选择数据分析相关技能

效果：当群 123456789 有人上传 CSV 或 XLSX 文件时，机器人会收到包含文件 ID 的提示，并建议使用指定技能进行分析。

**场景2：所有群的 PDF 文件自动总结**

模板组配置：
- `group_ids`: （留空）
- `file_patterns`: `.*\.pdf$`
- `skills`: 选择文档总结技能

**场景3：多群多规则配置**

```
模板1: 群A → 所有文件 → 技能X
模板2: 群A、群B → PDF文件 → 技能Y
模板3: 所有群 → CSV文件 → 技能Z
```

上传文件时按顺序匹配：群A上传 CSV 会匹配模板1；群B上传 PDF 会匹配模板2。

## 技术说明

### NapCatQQ API 调用

本插件使用以下 NapCatQQ OneBot API：

- `get_group_root_files` - 获取群文件根目录
- `get_group_files_by_folder` - 获取群文件列表
- `get_group_file_url` - 获取群文件下载链接
- `get_private_file_url` - 获取私聊文件下载链接
- `get_group_member_list` - 获取群成员列表（插件侧分页与身份映射）
- `get_qun_album_list` - 获取群相册列表（支持 `attach_info` 游标）
- `get_group_album_media_list` - 获取群相册媒体（支持 `attach_info` 游标）
- `group_upload` (Notice 事件) - 监听群文件上传

### 文件 ID 说明

- **群文件**: 通过 `qq_file`（`source=group_file`）获取的文件 ID 可用于获取下载链接
- **群相册媒体**: `qq_file` 会直接返回 NapCat 提供的媒体 URL，不使用群文件下载接口
- **私聊文件**: 用户发送的文件会自动生成 file_id，可通过工具获取链接

## 安装

将本插件放入 AstrBot 的 `plugins` 目录即可。

## 注意事项

1. 确保 NapCatQQ 版本支持群文件相关接口
2. 机器人需要有访问群文件的权限
3. **文件下载链接有过期时间**（通常为几分钟到几小时），请及时使用
4. 私聊文件需要在当前会话中获取，无法访问历史文件
5. 自动处理功能会触发 LLM 调用，请注意 API 费用
